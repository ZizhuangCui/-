# 飞书多维表格 Webhook 同步方案

这个方案用于绕开企业自建应用审批：不使用 App ID / App Secret，也不调用飞书开放平台写表格接口，而是让项目把每日汇总数据 POST 到飞书多维表格自动化 Webhook，由飞书自动化新增记录。

## 适用场景

- 公司暂时不好审批企业自建应用权限。
- 需要把风险评论、高赞评论、点赞增长评论沉淀到飞书空间。
- 可以接受使用多维表格按日期分组，而不是普通电子表格每天一个 sheet。

## 飞书侧配置

1. 新建一个多维表格：`评论风险与高赞评论每日汇总`
2. 新建数据表：`评论汇总`
3. 建议字段如下：

| 字段名 | 类型 |
| --- | --- |
| 日期 | 日期 |
| 平台 | 文本或单选 |
| 监控账号 | 文本 |
| 内容ID | 文本 |
| 评论ID | 文本 |
| 评论用户 | 文本 |
| 评论时间 | 日期时间 |
| 评论内容 | 多行文本 |
| 风险词 | 文本 |
| 是否风险词 | 单选 |
| 是否今日新增 | 单选 |
| 点赞总数 | 数字 |
| 昨日点赞数 | 数字 |
| 点赞新增 | 数字 |
| 高亮类型 | 文本 |
| 高亮等级 | 单选：红色、橙色、普通 |
| 检测时间 | 日期时间 |

4. 建议视图：

- 今日高亮：筛选日期为今天，按高亮等级排序。
- 红色预警：筛选高亮等级为红色。
- 高赞非风险：筛选是否风险词为否，点赞总数较高。
- 点赞增长：筛选点赞新增大于阈值。
- 按日期分组：按日期字段分组。

5. 新建自动化：

- 触发条件：接收到 Webhook 时
- 动作：新增记录
- 将 Webhook 入参映射到上面的字段

## 项目侧配置

后续项目会读取本地配置文件，例如：

```md
# Feishu Bitable Webhook Local Config

Webhook URL: `https://...`
```

本地密钥配置文件不要提交到 GitHub。

## 当前数据来源

项目扫描完成后会生成每日汇总 CSV：

```text
data/reports/comment_daily_summary_YYYY-MM-DD.csv
```

这个 CSV 已经包含风险词评论、非风险高赞评论、点赞增长评论，以及高亮等级。
